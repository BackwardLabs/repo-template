# 기여 가이드

## 이슈 작성

1. 기존 이슈를 검색해 중복 여부를 확인합니다.
2. 목적에 맞는 이슈 템플릿을 사용합니다.
3. 여러 저장소에 영향을 주면 관련 저장소, 이슈, PR과 예상 병합 순서를 함께 적습니다.
4. 로그와 화면 캡처에서 토큰, 개인정보, 내부 URL 등 민감 정보를 제거합니다.

보안 취약점은 공개 이슈로 등록하지 마세요. 저장소에서 안내하는 비공개 보안 제보 채널을 사용합니다.

## 브랜치 이름

이슈 번호와 변경 목적이 드러나는 짧은 이름을 권장합니다.

```text
feat/123-short-summary
fix/456-short-summary
docs/789-short-summary
```

## Pull Request

- 하나의 PR에는 하나의 논리적 변경만 담습니다.
- 관련 이슈를 연결하고 변경 이유와 범위를 설명합니다.
- 실행한 테스트와 확인 결과를 기록합니다.
- 호환성, 데이터 변경, 권한, 배포 순서 등 리뷰어가 확인해야 할 위험을 적습니다.
- 여러 저장소의 변경이 함께 필요하면 모든 관련 PR과 안전한 병합 순서를 연결합니다.

## 리뷰 기준

리뷰어는 다음을 중심으로 확인합니다.

- 변경이 이슈의 목적과 완료 조건을 충족하는가
- 기존 동작과 하위 호환성을 불필요하게 깨뜨리지 않는가
- 테스트 또는 검증 근거가 충분한가
- 운영, 보안, 개인정보, 데이터 마이그레이션 영향이 설명되어 있는가
- 관련 문서와 다른 저장소의 변경이 함께 반영되어 있는가
