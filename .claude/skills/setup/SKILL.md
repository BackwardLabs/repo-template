# 저장소 에이전트 설정

이 스킬은 템플릿에서 새 저장소를 만든 뒤 사용자가 `/setup`을 명시적으로 호출했을 때 한 번 실행합니다. 저장소에서 확인할 수 있는 사실을 우선하고, 선택이 필요한 항목만 사용자에게 질문하세요.

## 1. 저장소 확인

- `README.md`, `CONTRIBUTING.md`, `TEMPLATE_SETUP.md`를 읽습니다.
- 언어와 런타임은 `package.json`, `pyproject.toml`, `go.mod`, `Cargo.toml` 등 실제 매니페스트에서 확인합니다.
- 설치, 실행, 린트, 타입 검사, 테스트 및 빌드 명령은 매니페스트, Makefile과 CI 설정에서 확인합니다.
- 주요 디렉터리와 테스트 파일 패턴을 확인합니다.
- 현재 Git 변경을 확인하고 기존 사용자 작업을 보존합니다.

## 2. `AGENTS.md` 초기화

- 프로젝트 목적, 주요 기술과 디렉터리의 `TODO`를 확인된 정보로 교체합니다.
- 명령어의 `TODO`를 실제 명령으로 교체하고 안전하게 실행 가능한 명령은 직접 검증합니다.
- 저장소에서 확인할 수 없는 정보는 추측하지 말고 `TODO`로 유지한 뒤 사용자에게 알립니다.
- 공통 규칙은 `AGENTS.md`에 한 번만 작성하고 `CLAUDE.md`에 복사하지 않습니다.

## 3. Claude Code 설정 검토

- `CLAUDE.md`가 `@AGENTS.md`를 가져오는지 확인합니다.
- `.claude/settings.json`의 재귀적인 비밀 파일 차단 규칙과 프로젝트 고유의 비밀 경로를 확인합니다. 기존 차단 규칙을 이유 없이 완화하지 않습니다.
- 경로별 규칙이 반복해서 필요하고 실제 경로가 확인된 경우에만 `.claude/rules/` 파일을 만듭니다.
- 훅, MCP 서버, 추가 권한과 자동 실행 명령은 사용자 승인 없이 추가하지 않습니다.

## 4. 검증 및 마무리

- `AGENTS.md`에 기록한 명령과 실제 프로젝트 설정이 일치하는지 다시 확인합니다.
- 실행한 검사, 남은 `TODO`, 생성하거나 수정한 규칙과 미검증 항목을 보고합니다.
- 초기화가 끝나면 이 일회성 스킬 디렉터리(`.claude/skills/setup/`)를 삭제할지 사용자에게 확인합니다.
