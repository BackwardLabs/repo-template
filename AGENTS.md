# 저장소 에이전트 가이드

이 파일은 이 저장소에서 작업하는 코딩 에이전트의 공통 지침 원본입니다. 도구별 파일에 같은 규칙을 복사하지 말고 이 파일을 참조하세요. 하위 디렉터리에 더 구체적인 `AGENTS.md`가 있으면 해당 범위에서는 그 지침을 함께 따릅니다.

## 프로젝트 정보

- 목적: `TODO: 이 프로젝트가 해결하는 문제를 한 문장으로 작성`
- 주요 기술: `TODO: 언어, 프레임워크, 런타임 작성`
- 주요 디렉터리: `TODO: 소스, 테스트, 문서 디렉터리와 역할 작성`

프로젝트를 생성한 뒤 실제 저장소에서 확인한 정보로 위 항목을 교체하세요. 확인할 수 없는 내용을 추측해서 채우지 마세요.

## 기준 문서

- 프로젝트 개요와 실행 방법: [README.md](README.md)
- 이슈, Linear 브랜치, PR 및 리뷰 방식: [CONTRIBUTING.md](CONTRIBUTING.md)
- 템플릿 초기 설정: [TEMPLATE_SETUP.md](TEMPLATE_SETUP.md)
- 이슈 및 PR 입력 형식: `.github/`

작업 전 관련 문서를 읽고, 같은 규칙을 여러 파일에 중복해서 추가하지 마세요.

## 명령어

`TODO` 항목은 실행 가능한 명령이 아닙니다. 새 저장소를 만든 뒤 매니페스트, CI 설정, 기존 문서에서 실제 명령을 확인하고 동작을 검증한 후 교체하세요.

- 의존성 설치: `TODO`
- 로컬 실행: `TODO`
- 린트: `TODO`
- 타입 검사: `TODO`
- 테스트: `TODO`
- 빌드: `TODO`

## 작업 원칙

- 변경 전에 관련 코드, 테스트, 문서와 현재 Git 상태를 확인합니다.
- 요청을 만족하는 가장 작은 범위만 변경하고 관련 없는 파일을 정리하거나 재구성하지 않습니다.
- 기존 구조, 이름, 스타일을 우선하며 새로운 의존성이나 추상화는 필요성이 확인된 경우에만 추가합니다.
- 여러 저장소에 영향을 주는 변경은 호환성, 의존 PR과 안전한 병합 순서를 문서화합니다.
- 기존 사용자 변경을 덮어쓰거나 삭제하지 않습니다.

## 검증 및 완료 기준

- 변경과 직접 관련된 검사를 먼저 실행하고, 완료 전 설정된 린트, 타입 검사, 테스트 및 빌드 중 해당하는 항목을 실행합니다.
- 버그 수정에는 가능하면 실패를 재현하는 테스트 또는 검증 절차를 포함합니다.
- 실행할 수 없는 검사가 있으면 생략하지 말고 이유와 미검증 범위를 결과에 명시합니다.
- 코드, 테스트, 문서와 PR 설명이 서로 일치할 때 작업을 완료합니다.

## GitHub 및 Linear

- 브랜치와 PR 규칙은 [CONTRIBUTING.md](CONTRIBUTING.md)를 따릅니다.
- Linear 이슈의 **Copy git branch name**으로 브랜치를 만들고 이슈 ID를 유지합니다.
- 한 PR에는 하나의 논리적 변경만 담고, 관련 이슈와 다른 저장소의 의존 PR을 연결합니다.
- 커밋 메시지는 Conventional Commits 형식의 영어 명령문으로 작성합니다.

## 보안

- 비밀 정보, 토큰, 개인정보, 내부 URL을 코드, 로그, 이슈 또는 커밋에 포함하지 않습니다.
- `.env`와 로컬 설정 파일을 읽거나 커밋하지 않습니다. 공유 가능한 값은 `.env.example`에 이름과 설명만 남깁니다.
- 위험하거나 되돌리기 어려운 작업, 외부 시스템 변경, 권한 확대는 실행 전에 명시적인 승인을 받습니다.
